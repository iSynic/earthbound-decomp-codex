## Good-Enough Boundary

For this milestone, text/script assets are good enough when the project can
separate these buckets cleanly:

- live runtime commands with documented handler semantics
- parser-only compressed-bank pseudo-opcodes
- parser artifacts caused by table, pointer, or compressed-data overlap
- recovered localization authoring commands that are metadata/macros rather
  than direct ROM text VM opcodes

That boundary lets future extraction or reinsertion tooling avoid treating
every unmatched byte as a new text VM feature.

For the fuller recovered-source authoring syntax view, see
`notes/localization-authoring-command-frontier.md`.
