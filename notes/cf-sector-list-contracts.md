# CF Sector List Contracts

Generated by `tools/build_cf_sector_list_contracts.py` from `src/cf/bank_cf_helpers_asar.asm` and `src/d0/bank_d0_helpers_asar.asm`.

## Main result

The CF counted sector-list families now have reproducible subrecord boundaries:

- `DOOR_CONFIG_TABLE` covers `CF:264F..CF:58EE` as `1280` D0-pointer-addressed counted sector lists.
- In source-order physical row view, those lists contain `2080` five-byte movement-trigger rows across `601` non-empty sectors, matching the existing `map_doors.yml` bundle count.
- In raw pointer-consumer view, the same starts expose `2100` candidate rows because `19` pointer starts overlap the previous counted-list tail by `100` total bytes.
- `D0_DOOR_POINTER_TABLE` covers `D0:0000..D0:13FF`; all `1280` long pointers target `CF:264F..CF:58EE` with bank byte `CF` and zero reserved byte.
- `SPRITE_PLACEMENT_POINTER_TABLE` covers `CF:61E7..CF:6BE6` with `627` nonzero pointers and `653` empty-sector sentinels.
- `SPRITE_PLACEMENT_TABLE` covers `CF:6BE7..CF:8984` as `627` counted lists with `1582` four-byte placement rows.
